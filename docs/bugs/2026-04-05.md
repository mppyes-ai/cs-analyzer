# CS-Analyzer Bug记录

## BUG-2026-04-05-001: API超时硬编码

**状态**: 待修复  
**优先级**: P2  
**发现日期**: 2026-04-05

### 问题描述

`smart_scoring_v2.py` 中 `_call_kimi_async()` 函数的 timeout 被硬编码为300秒，`.env` 中的 `KIMI_API_TIMEOUT` 配置无效。

### 代码位置

```python
# smart_scoring_v2.py ~680行
async def _call_kimi_async(self, prompt: str, expected_count: int, pre_analyses: List[Dict] = None) -> List[Dict]:
    timeout_seconds = 300  # 【v2.5】硬编码 300 秒 ← 问题在这里！
    # 虽然前面有 os.getenv，但这行覆盖了配置
```

### 影响

- `.env` 中 `KIMI_API_TIMEOUT` 配置无效
- 无法灵活调整API超时时间

### 修复方案

```python
timeout_seconds = float(os.getenv('KIMI_API_TIMEOUT', '300'))
```

## BUG-2026-04-05-002: 批量评分API返回截断

**状态**: 已修复  
**优先级**: P1  
**发现日期**: 2026-04-05

### 问题描述

当批量评分超过20通时，Kimi API返回的结果被截断，只有部分会话有评分数据，其余为0分。

### 根因分析

1. `KIMI_MAX_TOKENS=16000` 不足以支持50通会话的完整评分输出
2. `calculate_adaptive_batch_size()` 计算出的批量大小可达50通，超出API返回长度限制

### 修复方案

1. 限制最大批量为20通：`min(potential_size, 20, len(sessions))`
2. 提升Token上限：`KIMI_MAX_TOKENS=32000`

### 相关文件

- `worker.py`: `calculate_adaptive_batch_size()`
- `.env`: `KIMI_MAX_TOKENS`
