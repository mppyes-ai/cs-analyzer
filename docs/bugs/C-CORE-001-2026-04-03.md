## 问题描述

**现象**: 在并行模式 (`--parallel`) 下执行分析任务时，Worker进程崩溃，抛出 `NameError: name 'is_transfer' is not defined`

**影响范围**: 
- 仅影响 `--parallel` 模式
- 串行模式 (`--serial`) 和分组模式 (`--grouped`) 不受影响
- 导致并行分析完全无法使用

## 根因分析

**问题位置**: `worker.py:736` `_save_result_parallel()` 函数

**代码缺陷**:
```python
def _save_result_parallel(session_id: str, session_data: dict, intent, result: dict, total: int):
    """并行模式：保存分析结果（线程安全版本）"""
    # ... 前面代码 ...
    
    cursor.execute('''
        INSERT OR REPLACE INTO sessions 
        (session_id, user_id, staff_name, messages, summary, 
         professionalism_score, standardization_score, policy_execution_score, conversion_score,
         total_score, analysis_json, strengths, issues, suggestions, session_count, start_time, end_time, created_at,
         is_transfer, transfer_from, transfer_to, transfer_reason, related_sessions)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, 1, ?, ?, ?, ?, ?, ?, ?, ?)
    ''', (
        # ... 其他参数 ...
        1 if is_transfer else 0,  # ❌ NameError: is_transfer 未定义
        transfer_from,            # ❌ NameError: transfer_from 未定义
        transfer_to,              # ❌ NameError: transfer_to 未定义
        transfer_reason,          # ❌ NameError: transfer_reason 未定义
        json.dumps(related_sessions, ensure_ascii=False)  # ❌ NameError: related_sessions 未定义
    ))
```

**原因**: 
`_save_result_parallel()` 函数在保存结果时，使用了5个转接相关变量，但这些变量从未在函数内定义或从 `session_data` 提取。

对比串行模式的 `save_to_database()` 函数，这些变量是正确从 `session_data` 提取的。
