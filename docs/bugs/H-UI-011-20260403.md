# Bug H-UI-011: Streamlit前端预分析显示"N/A"

**状态**: ✅ 已修复  
**修复日期**: 2026-04-03  
**严重级别**: 中（影响数据展示，不影响核心评分功能）  
**影响版本**: v2.4 async-batch模式

---

## 问题描述

Streamlit前端「会话分析与矫正」页面中，会话明细下的**预分析信息**（场景/意图/情绪/置信度）均显示"N/A"。

## 根因分析

**代码路径**: `smart_scoring_v2.py`

- **单通评分** (`score_session`方法): ✅ 正确存储 `_metadata.pre_analysis`
- **批量评分** (`score_sessions_batch_async`方法): ❌ **缺失** `pre_analysis` 参数传递和存储逻辑

具体原因：
1. `_score_batch_same_scene` 调用 `_call_kimi_async` 时未传递 `pre_analyses`
2. `_call_kimi_async` 方法签名中没有 `pre_analyses` 参数
3. 评分结果组装时没有将预分析数据存入 `_metadata.pre_analysis`

## 修复方案

### 文件变更
- `skills/cs-analyzer/smart_scoring_v2.py` (2处修改)

### 代码变更

**第1处: `_score_batch_same_scene` 方法**
```diff
- result = await self._call_kimi_async(prompt, len(sessions))
+ result = await self._call_kimi_async(prompt, len(sessions), pre_analyses)
```

**第2处: `_call_kimi_async` 方法**
```diff
- async def _call_kimi_async(self, prompt: str, expected_count: int) -> List[Dict]:
+ async def _call_kimi_async(self, prompt: str, expected_count: int, pre_analyses: List[Dict] = None) -> List[Dict]:
      ...
      # 补充元数据（包含预分析数据）
      for i, r in enumerate(results):
          if '_metadata' not in r:
              r['_metadata'] = {}
          r['_metadata']['model'] = model
          r['_metadata']['scored_at'] = datetime.now().isoformat()
+         # 添加预分析数据（从pre_analyses获取）
+         if pre_analyses and i < len(pre_analyses):
+             r['_metadata']['pre_analysis'] = pre_analyses[i]
```
