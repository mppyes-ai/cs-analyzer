## CS-Analyzer 完整闭环链路详解

### 一、数据接收阶段

#### 1.1 输入来源
```
┌─────────────────────────────────────────┐
│           数据来源                        │
├─────────────────────────────────────────┤
│ 1. 客服聊天记录文件 (.log/.txt)          │
│ 2. 实时消息流 (WebSocket/API)            │
│ 3. 历史数据导入 (CSV/JSON)               │
│ 4. 飞书消息同步                          │
└─────────────────────────────────────────┘
```

#### 1.2 数据格式示例
```
[2025-04-27 10:00:00] 用户A: 你好，我想买热水器
[2025-04-27 10:00:05] 客服B: 您好，请问有什么可以帮您？
[2025-04-27 10:00:10] 用户A: 林内13升的价格多少？
[2025-04-27 10:00:15] 客服B: 这款目前活动价2999元
```

#### 1.3 数据接收处理
```python
# cs_analyzer_batch.py
1. 文件上传 → 验证格式
2. 编码检测 → UTF-8/GBK转换
3. 文件大小检查 → 限制500MB
4. 存储到临时目录 → /tmp/cs_analyzer/
```

### 二、数据解析阶段

#### 2.1 日志解析流程
```
原始日志文件
    ↓
LogParser.parse_log_file()
    ↓
├─ 时间戳提取 → 2025-04-27 10:00:00
├─ 用户ID提取 → 用户A
├─ 客服ID提取 → 客服B
├─ 消息内容提取 → "你好，我想买热水器"
└─ 会话边界识别 → 30分钟无消息视为新会话
```

#### 2.2 会话分组逻辑
```python
# 按用户ID分组
sessions = {
    "用户A": [
        {"time": "10:00:00", "role": "user", "content": "你好..."},
        {"time": "10:00:05", "role": "staff", "content": "您好..."},
        # ...
    ]
}
```

#### 2.3 数据清洗
```
├─ 过滤无用户消息的会话 → 删除
├─ 过滤超短会话 (<2轮对话) → 删除
├─ 过滤测试数据 → 标记
└─ 敏感信息脱敏 → 手机号/地址替换
```

### 三、会话分析阶段

#### 3.1 分析流程图
```
┌─────────────────────────────────────────┐
│           会话分析引擎                    │
├─────────────────────────────────────────┤
│                                         │
│  会话数据 → 预分析 → 规则检索 → LLM评分  │
│     ↓        ↓        ↓        ↓       │
│  结构化   场景识别   知识库    四维度   │
│  数据     意图识别   匹配      评分     │
│  情绪识别   规则筛选              │
│                                         │
└─────────────────────────────────────────┘
```

#### 3.2 预分析模块
```python
# smart_scoring_v2.py _analyze_session_pre()

1. 场景识别
   ├─ 关键词匹配 → "多少钱" → 售前咨询
   ├─ 意图识别 → "投诉" → 客诉处理
   └─ 情绪识别 → "骗子" → negative

2. 结构化数据
   {
       "scene": "售前咨询",
       "sub_scene": "价格咨询",
       "intent": "咨询",
       "sentiment": "neutral",
       "confidence": 0.85
   }
```

#### 3.3 规则检索模块
```python
# hybrid_retriever.py HybridRuleRetriever

1. 全文检索 (40%权重)
   ├─ SQLite LIKE匹配
   └─ 触发关键词匹配

2. 向量检索 (60%权重)
   ├─ 会话文本 → Embedding模型 → 向量
   ├─ LanceDB相似度搜索
   └─ 返回Top-K相关规则

3. 混合排序
   ├─ 加权合并分数
   └─ Rerank返回最终规则列表
```

#### 3.4 LLM评分模块
```python
# smart_scoring_v2.py score_session()

输入:
├─ SYSTEM_PROMPT_V2 (固定)
├─ FIXED_USER_PREFIX (缓存优化)
├─ 检索到的规则 (动态)
└─ 会话数据 (动态)

输出:
{
    "dimension_scores": {
        "professionalism": {"score": 4, "reason": "..."},
        "standardization": {"score": 3, "reason": "..."},
        "policy_execution": {"score": 5, "reason": "..."},
        "conversion": {"score": 2, "reason": "..."}
    },
    "total_score": 14,
    "risk_level": "medium"
}
```

#### 3.5 评分维度详解
| 维度 | 说明 | 评分标准 |
|------|------|---------|
| **专业性** | 产品知识、解答能力 | 5分:专业详尽 3分:基本正确 1分:错误 |
| **标准化** | 话术规范、流程执行 | 5分:完全规范 3分:基本规范 1分:违规 |
| **政策执行** | 活动规则、售后政策 | 5分:准确执行 3分:部分正确 1分:错误 |
| **转化能力** | 销售技巧、促成下单 | 5分:主动促成 3分:被动回答 1分:无转化 |

### 九、数据流向图

```
客服聊天记录(400).log
    ↓
[LogParser] 解析 → sessions.jsonl
    ↓
[TaskQueue] 分组 → 88个任务
    ↓
[Worker] 异步处理
    ├─ [IntentClassifier] 场景识别
    ├─ [HybridRetriever] 规则检索 (SQLite + LanceDB)
    ├─ [SmartScoring] LLM评分 (oMLX本地模型)
    └─ [DBWriter] 结果存储
        ↓
    SQLite: sessions表 + analysis_results表
    LanceDB: 规则向量索引
    飞书: 进度通知 + 完成报告
        ↓
[人工复核] 矫正反馈
    ↓
[RuleExtractor] 自动提取规则
    ↓
[知识库] 审批入库 → 同步向量库
    ↓
[下次评分] 使用新规则
```

|------|------|------|
