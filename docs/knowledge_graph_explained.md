## 一、什么是知识图谱？

### 1.1 通俗理解

**传统数据库** = Excel表格（行列结构）
**知识图谱** = 大脑神经网络（节点+连接）

```
传统数据库（Excel思维）
┌─────────┬─────────┬─────────┐
│ 产品型号 │ 价格    │ 容量    │
├─────────┼─────────┼─────────┤
│ GD31    │ 2599    │ 13L     │
│ GD32    │ 3199    │ 16L     │
└─────────┴─────────┴─────────┘
问题：GD32和GD31什么关系？→ 需要另一张表

知识图谱（大脑思维）
    [GD32] ──升级版本──→ [GD31]
      │                    │
   适用政策             适用政策
      │                    │
   [安装政策]          [安装政策]
      │                    │
   涉及场景             涉及场景
      │                    │
   [售前咨询]          [售前咨询]
      │                    │
   关联问题             关联问题
      │                    │
   [优惠券]            [优惠券]
```

### 1.2 核心概念

| 概念 | 说明 | 璞康示例 |
|------|------|---------|
| **实体（节点）** | 业务对象 | 产品、客户、场景、政策 |
| **关系（边）** | 对象间联系 | 适用、关联、升级、替代 |
| **属性** | 对象特征 | 价格、容量、功能 |
| **图谱** | 实体+关系的网络 | 璞康业务知识网络 |

## 三、知识图谱 vs 传统数据库

### 3.1 查询对比

**场景：用户问"GD32和GD31有什么区别？"**

```sql
-- 传统数据库：需要多张表JOIN
SELECT p1.型号, p1.容量, p1.价格, p2.型号, p2.容量, p2.价格
FROM 产品表 p1
JOIN 产品表 p2 ON p1.对比型号 = p2.型号
WHERE p1.型号 = 'GD32' AND p2.型号 = 'GD31';
-- 结果：只有基础参数，没有功能差异、适用场景

-- 知识图谱：直接遍历关系
MATCH (gd32:产品)-[:升级版本]->(gd31:产品)
RETURN gd32, gd31,
       gd32.核心功能 AS gd32功能,
       gd31.核心功能 AS gd31功能,
       gd32.适用场景 AS gd32场景,
       gd31.适用场景 AS gd31场景,
       gd32.价格 - gd31.价格 AS 差价;
-- 结果：完整对比，包含功能、场景、价格差异
```

### 3.2 推理对比

**场景："我家一厨两卫，推荐哪款？"**

```sql
-- 传统数据库：无法推理
SELECT * FROM 产品表 WHERE 容量 >= 16L;
-- 结果：只返回产品列表，没有推荐理由

-- 知识图谱：基于关系推理
MATCH (c:客户 {家庭情况: "一厨两卫"})-[:咨询过]->(s:场景)
MATCH (s)-[:推荐产品]->(p:产品)
WHERE p.容量 >= 16L
RETURN p,
       p.核心功能 AS 推荐理由,
       p.价格 AS 价格,
       (p)-[:适用政策]->(po:政策) AS 可用政策;
-- 结果：推荐GD32，理由：水量伺服器适合多点用水，有国补政策
```

## 四、璞康多场景应用

### 4.1 场景1：客服质检（当前）

```python
# 输入：客服会话
session = {
    "用户问题": "GD32和GD31有什么区别？",
    "客服回答": "GD32是16L，GD31是13L，GD32多了水量伺服器"
}

# 知识图谱查询
MATCH (s:场景 {名称: "售前咨询"})-[:评分标准]->(r:规则)
WHERE s.触发词 CONTAINS "区别"
RETURN r.维度, r.标准, r.示例;

# 结果：
# 维度：专业性
# 标准：需对比核心参数（容量、功能）
# 示例：优秀回答应包含水量伺服器差异
# 评分：4分（缺少恒温技术解释）
```

### 4.2 场景2：产品培训（未来）

```python
# 输入：新员工培训
 trainee = "需要了解GD32产品知识"

# 知识图谱查询
MATCH (p:产品 {型号: "GD32"})
OPTIONAL MATCH (p)-[:升级版本]->(p2:产品)
OPTIONAL MATCH (p)-[:适用政策]->(po:政策)
OPTIONAL MATCH (p)-[:常见咨询]->(s:场景)
RETURN p.核心功能 AS 产品卖点,
       p2.型号 AS 对比型号,
       po.名称 AS 促销政策,
       s.触发词 AS 常见问题;

# 结果：生成培训材料
# - 产品卖点：水量伺服器、恒温精准
# - 对比型号：GD31（差价600元）
# - 促销政策：国补20%（补贴后2559元）
# - 常见问题：价格、安装尺寸、优惠券
```

### 4.3 场景3：经营分析（未来）


### 4.4 场景4：项目诊断（未来）

```python
# 输入：京东店铺诊断
diagnosis = "页面转化率低于行业平均"

# 知识图谱查询
MATCH (p:页面 {类型: "商详页"})
OPTIONAL MATCH (p)-[:展示]->(pr:产品)
OPTIONAL MATCH (pr)-[:常见咨询]->(s:场景)
OPTIONAL MATCH (s)-[:关联问题]->(q:问题)

RETURN pr.型号 AS 产品,
       q.内容 AS 页面问题,

# - 产品：GD32

#   1. "主图看不清功能"（高优先级）
#   2. "没有安装尺寸图"（中优先级）
```

## 六、金总，总结

### 6.1 核心差异

| 维度 | 传统数据库 | 知识图谱 |
|------|----------|---------|
| **数据结构** | 表格（行列） | 网络（节点+边） |
| **查询方式** | 条件筛选 | 关系遍历 |
| **推理能力** | 无 | 基于路径推理 |
| **扩展性** | 需改表结构 | 动态添加实体 |
| **应用场景** | 单一 | 多场景复用 |

### 6.2 璞康价值

```
当前：客服质检规则库
    ↓ 升级
目标：企业知识中枢
    ├─ 客服质检（已验证）
    ├─ 产品培训（新员工上手）
    ├─ 经营分析（数据驱动决策）
    └─ 项目诊断（问题根因分析）
```

### 6.3 实施建议

**Phase 1（1-2月）：核心图谱构建**
- 定义实体类型：产品、场景、政策、客户
- 迁移现有规则为图谱节点
- 建立基础关系网络

**Phase 2（2-3月）：多场景接入**
- 客服质检（优化现有）
- 产品培训（生成课程）

**Phase 3（3-4月）：深度应用**
- 经营分析（关联销售数据）
- 项目诊断（对比竞品数据）
