## 技术根因分析

### 根因1: 会话重复处理（批次77）

**机制推测**:
```
用户多次发送消息 → 生成多个task_id → 同一session_id
→ Worker按user_id分组 → 重复session_id进入同一批次
→ 模型内部发现重复 → 触发去重/合并逻辑
→ 增加推理复杂度 → API耗时232.6s
```

**为何未被去重**:
- 当前Worker按`user_id`分组，未检测`session_id`重复
- 任务164和172的task_id不同，被视为独立任务
- 模型层才感知到重复，但已消耗API调用

### 根因2: 客诉场景复杂度（批次46）

**机制推测**:
```
用户情绪激动 + 多轮交锋 → 30条消息
→ Token量达12K+ → 接近批次上限
→ 模型需要复杂情感分析 → 评分维度冲突（policy_execution=0）
→ 增加推理时间 → API耗时189.0s
```

**评分异常**:
```
⚠️ policy_execution分数截断: 0 -> 1
⚠️ conversion分数截断: 0 -> 1
```
- 模型对客诉场景给出极低分数，触发截断保护
- 表明场景超出模型正常评分范围

## 优化建议

### P0: 去重前置（立即实施）

**目标**: 消除批次77的重复会话问题，节省65%耗时

**实现方案**:
```python
def deduplicate_sessions(tasks: List[Dict]) -> List[Dict]:
    """在送入模型前去重，保留最早的任务"""
    seen_sessions = {}
    for task in sorted(tasks, key=lambda x: x['created_at']):
        session_id = task['session_id']
        if session_id in seen_sessions:
            # 标记重复任务为完成，不送模型
            cancel_task(task['task_id'], reason="Duplicate session")
        else:
            seen_sessions[session_id] = task['task_id']
    return [t for t in tasks if t['session_id'] in seen_sessions]

# 在prepare_merged_tasks_sync中调用
merged_tasks = deduplicate_sessions(raw_tasks)
```

**预期效果**:
- 批次77 API耗时: 232.6s → ~140s
- 400通总耗时: ~12分钟 → ~10分钟

### P1: 客诉场景降级（中期实施）

**目标**: 降低批次46的高复杂度会话影响

**实现方案**:
```python
def detect_complaint_scene(messages: List[Dict]) -> bool:
    """检测客诉场景关键词"""
    complaint_keywords = [
        "抱歉有啥用", "催", "投诉", "诚信在哪",
        "你烦我也烦", "一遍一遍", "都.*个月了"
    ]
    content = ' '.join([m.get('content', '') for m in messages])
    return any(re.search(kw, content) for kw in complaint_keywords)

def adaptive_batch_size(tasks: List[Dict]) -> int:
    """客诉场景单通处理"""
    for task in tasks:
        if detect_complaint_scene(task['messages']):
            return 1  # 强制单通
    return 5  # 默认5通
```

**预期效果**:
- 批次46 API耗时: 189.0s → ~50s（单通）
- 避免客诉会话影响其他正常会话

### P2: Token预估优化（长期实施）

**目标**: 动态调整批次大小，避免超长会话

**实现方案**:
```python
MAX_TOKENS_PER_TASK = 10000

def calculate_batch_size(tasks: List[Dict]) -> int:
    """基于Token预估动态调整"""
    total_tokens = sum(estimate_tokens(t) for t in tasks)
    if any(estimate_tokens(t) > MAX_TOKENS_PER_TASK for t in tasks):
        return 1  # 超长会话单通
    return min(5, len(tasks))
```
