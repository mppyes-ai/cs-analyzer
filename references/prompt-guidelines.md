# CS-Analyzer Prompt 设计规范

**文档版本**: 2026-03-22  
**适用范围**: 客服会话质量分析系统 v2

---

## 1. 评分输出格式规范

### 1.1 reasoning字段

**要求**: 直接陈述事实和评分依据，不要加任何前缀。

**❌ 禁止格式**:
```
判定过程：基于checkpoints分析：
Checkpoint 1: xxx
Checkpoint 2: xxx
```

**✅ 正确格式**:
```
客服对安装辅材的回答基本准确，说明具备基础产品知识；但客服在未确认用户购买状态的情况下...
```

**清理规则**（历史数据）:
- 去掉"判定过程："
- 去掉"Checkpoint X：" / "Checkpoints:"
- 去掉数字列表符号"1)" "2)"、"-"列表符号
- 去掉"分析："、"结论："等前缀

## 2. 规则引用规范

### 2.1 referenced_rules字段

| 场景 | 值 | 说明 |
|------|-----|------|
| 命中知识库规则 | `["rule_xxx"]` | 填写具体规则ID |
| 知识库未覆盖 | `[]` | 空数组表示基于行业通用标准评判 |

### 2.2 通用标准定义

当知识库未覆盖时，AI基于以下行业通用标准评判：

- **电商行业通用标准**: 响应时效、服务态度、问题闭环等电商客服基本规范
- **客服服务通用标准**: 首响及时、礼貌用语、主动解决、结束规范等服务行业基本要求
- **消费者认可标准**: 信息准确、不推诿、有效解决用户问题等消费者普遍期望

**注意**: 这些通用标准未被审核通过前，**不能**作为知识库规则显示在"规则命中总览"中。

## 3. 前端显示对应关系

### 3.1 判定过程背景色

```python
is_generic = len(referenced_rules) == 0
```

| 条件 | 背景色 | 含义 |
|------|--------|------|
| `is_generic == True` | 🟡 黄色 | 使用行业通用标准 |
| `is_generic == False` | ⚪ 灰色 | 命中知识库规则 |

### 3.2 规则命中总览

**无知识库规则时**:
```
📭 未命中知识库规则，基于通用标准评判（4/4个维度使用通用标准）
```

**有部分规则时**:
```
[显示知识库规则列表]
💡 注：2/4个维度使用通用标准评判
```

## 4. Prompt模板关键片段

### 4.1 评分要求

```
4. **输出推理过程**：用自然语言描述为什么给这个分数，直接陈述事实和依据，不要加"判定过程："等前缀
```

### 4.2 引用规则说明

```
### 引用规则说明
1. **使用知识库规则时**：`referenced_rules`填写具体的规则ID（如：`["rule_xxx"]`）
2. **知识库未覆盖时**：`referenced_rules`填写空数组（`[]`），表示基于电商/客服/售后等行业通用标准评判

### 通用标准定义
- **电商行业通用标准**：响应时效、服务态度、问题闭环等电商客服基本规范
- **客服服务通用标准**：首响及时、礼貌用语、主动解决、结束规范等服务行业基本要求
- **消费者认可标准**：信息准确、不推诿、有效解决用户问题等消费者普遍期望
```

### 4.3 reasoning示例

```json
{
  "professionalism": {
    "score": 3,
    "reasoning": "用户询问产品参数，客服回答准确但缺少对比说明，符合3分标准",
    "referenced_rules": ["rule_xxx"]
  },
  "standardization": {
    "score": 2,
    "reasoning": "响应及时但礼貌用语不够规范，存在服务瑕疵",
    "referenced_rules": []  // 知识库未覆盖
  }
}
```
